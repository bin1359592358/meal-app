# 分享与厨师订单通知设计

## 目标

为“超大一碗饭”增加两类微信好友分享能力：

1. 用户把小程序或当前餐桌分享给好友，好友点击后直接进入指定餐桌的邀请加入流程，无需手动输入邀请码。
2. 用户下单成功后，通过微信分享卡片手动通知厨师；厨师点击卡片后直接进入对应订单的处理页面。

微信小程序不能由后台自动向指定联系人发送普通分享卡片，因此通知厨师采用用户主动点击 `open-type="share"` 后选择微信会话的方式，不引入订阅消息模板。

## 视觉方案

沿用现有蓝白奶油色轻二次元风格，以原头像中的白发蓝眼厨娘为身份参考，但分享插画使用正常人物比例，不使用大头小人造型。

生成并使用两张 5:4 分享资源：

- `order-notify.png`：厨娘递出订单小票，左侧叠加“已下单，请快速处理！”。
- `invite-friends.png`：厨娘端着一大碗饭，左侧叠加“来和我一起点菜吧！”。

下单成功弹层复用订单通知插画，包含“通知厨师处理”和“稍后再说，查看订单”两个操作。

## 功能设计

### 普通分享

- 菜单页启用右上角微信分享。
- 个人页保留并强化“邀请好友一起点菜”入口。
- 餐桌详情同时提供两种邀请方式：
  - “复制邀请码”：保留现有邀请码，适合线下口述或跨平台发送。
  - “直接邀请好友”：使用 `button open-type="share"` 打开微信好友选择，免去复制和输入邀请码。
- 有当前餐桌时，分享路径为 `/pages/login/login?inviteCode=<餐桌邀请码>`。
- 没有当前餐桌时，分享路径为 `/pages/login/login`。
- 分享标题优先使用餐桌名，例如：`「周末聚餐」邀请你一起点菜`。
- 分享图片使用 `invite-friends.png`。

### 好友直接加入餐桌

- 好友点击餐桌邀请卡片后，小程序读取 `inviteCode`。
- 未登录时先完成微信登录，登录后自动继续处理该邀请码。
- 已登录时直接查询餐桌信息，展示餐桌名称、主人和“加入餐桌”确认按钮。
- 用户确认后调用现有加入餐桌接口，成功后将该餐桌设为当前餐桌并进入菜单页。
- 已经是餐桌成员时不重复加入，直接切换到该餐桌并进入菜单页。
- 邀请码失效、餐桌关闭或成员已满时显示对应原因，并允许返回个人页选择其他餐桌。

### 下单成功通知

- 订单创建成功后不再立即跳转。
- 购物车清空，并展示自定义成功弹层。
- “通知厨师处理”使用 `button open-type="share"` 调起微信分享面板。
- 分享标题：`新订单已提交，请厨师大人快速处理～`。
- 分享路径：`/pages/order-detail/order-detail?id=<订单ID>&from=chefShare`。
- 分享图片使用 `order-notify.png`。
- 点击“稍后再说，查看订单”进入订单详情；订单接口没有返回 ID 时进入订单列表。

### 订单详情再次通知

- 订单详情页增加“通知厨师处理”分享按钮。
- 任何能够查看该订单的餐桌成员都可手动分享，但厨师仍需具备原有订单管理权限才能修改状态。
- 分享按钮点击后显示轻提示“请选择厨师的微信会话”。
- 本地记录该订单是否点击过通知按钮，已点击后文案变为“再次通知厨师”，不限制再次分享。

### 厨师打开卡片

- 已登录且具有餐桌权限：直接打开订单详情，并显示现有状态处理控件。
- 未登录：沿用登录流程；登录完成后恢复订单详情目标地址。
- 无权限或订单不存在：显示明确提示并返回订单列表，不暴露订单内容。

## 模块边界

- `utils/share.js`：集中生成普通分享和订单通知的标题、路径、图片地址。
- 餐桌详情/个人页：提供“复制邀请码”和“直接邀请好友”两个并列入口。
- 购物车页：只负责订单成功弹层和当前订单分享上下文。
- 订单详情页：只负责再次通知入口和订单展示。
- 登录页：保存并恢复分享进入时的目标路径。
- 后端：沿用现有订单权限与接口，本阶段不新增消息推送接口。

## 状态与异常处理

- 下单提交期间继续防止重复点击。
- 订单成功但分享被取消，不影响订单创建与购物车清空。
- 分享功能不可用时仍可进入订单详情。
- 缺少订单 ID 时禁用订单通知分享，回退到订单列表。
- 分享卡片不包含菜品明细、备注等隐私信息，只显示通用提醒；详情需登录并通过后端权限校验后查看。

## 验证

- 单元测试覆盖分享标题、邀请码路径、直接加入餐桌、重复成员回退、订单路径及缺失参数回退。
- 微信开发者工具验证普通分享、下单成功弹层、订单详情再次分享三个入口。
- 手机预览验证微信分享面板能够打开，卡片图片与标题正确，厨师点击后进入目标订单。
- 回归现有购物车清空、防重复下单、订单详情权限和 TabBar 流程。
